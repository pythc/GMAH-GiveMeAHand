# 安全设计

## 基线原则

- 最小权限：工具、MCP server、渠道 adapter 都必须按 scope 授权。
- 高风险动作审批：成绩发布、外部通知、数据删除、批量写入等必须人工审批。
- 幂等与重放保护：写操作必须有幂等键和重复提交处理。
- 审计可追溯：工具调用、审批、外发消息、记忆写入都必须记录来源。
- 敏感信息保护：配置不硬编码密钥，日志不输出 token、密码或个人隐私。

## 工具风险等级

| 等级 | 示例 | 策略 |
|---|---|---|
| low | 查询公开资源、读取 rubric | 可自动执行，记录审计 |
| medium | 保存草稿、读取学生提交 | 策略审批或权限校验 |
| high | 发布成绩、外发通知、删除数据 | 人工审批 + 幂等 + 审计 |

## 记忆安全

- 不把未经确认的用户输入直接写入长期记忆。
- 记忆条目必须记录来源、时间、scope、置信度和敏感标记。
- 对冲突记忆采用版本化和失效策略。
- 提示注入内容不得升级为系统规则或长期偏好。

## MCP 安全

- 只允许配置白名单内 MCP server。
- 每个 server 需要独立 scope、timeout 和 allowlist。
- 不把密钥放进 tool JSON 或模型上下文。
- 对 remote MCP 的身份、来源和权限做显式验证。

## 数据库安全

- 必须使用参数化查询。
- 动态字段必须白名单校验。
- 禁止把用户输入拼接进 SQL。
