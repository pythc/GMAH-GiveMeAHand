# 记忆与上下文管理

## 分层记忆

| 层级 | 存储 | 内容 | 生命周期 | 写入策略 |
|---|---|---|---|---|
| 工作记忆 | LangGraph state / Redis | 当前消息、临时变量、工具结果 | 秒到小时 | 自动写入 |
| 会话摘要 | Redis / Postgres / Markdown | 决策、待办、引用、工具副作用 | 小时到周 | 触发式摘要 |
| Episodic memory | 向量库 + 元数据 | 某次批改、沟通、任务事件 | 周到月 | 事件提炼后写入 |
| Semantic memory | `MEMORY.md` + Postgres | 稳定事实、偏好、规则 | 长期 | 人审或高置信写入 |
| 身份记忆 | `AGENTS.md` / `SOUL.md` | 行为边界、风格、价值约束 | 长期 | 人工维护 |

## 文件规范

- `AGENTS.md`：权威项目/代理工作规范。
- `AGENT.md`：兼容副本模板。
- `agent.md`：小写兼容模板，仅在历史工具链需要时使用。
- `SOUL.md`：人格、风格、价值边界，不放易过期业务事实。
- `MEMORY.md`：稳定事实与偏好，必须记录来源和写入策略。

## 摘要触发条件

- 当前上下文超过模型窗口的 50% 到 60%。
- 连续对话达到 12 到 20 轮。
- 工具链产生大量结构化结果。
- 会话 idle 超过 30 分钟后恢复。
- 渠道切换，例如 Web 切到 QQ。

## 防污染策略

- 默认不把用户输入直接写入长期记忆。
- 记忆候选必须包含来源、置信度、时间和 scope。
- 互相矛盾的记忆需要人工确认或失效旧版本。
- 任何敏感信息写入前必须脱敏或拒绝写入。
